---
name: Runtime Permission Primer System
description: Unified soft-prompt → OS-prompt flow for mic, camera, photos, notifications via usePermissionPrimer + PermissionPrimerDialog
type: feature
---
All sensitive permissions (microphone, camera, photos, notifications) use
the unified `usePermissionPrimer()` hook + `<PermissionPrimerDialog />` from
`src/components/permissions/`. Flow:

1. Feature-trigger calls `await ensurePermission(kind)`.
2. If state is `granted` → returns true immediately.
3. If `denied` or previously suppressed → toast hint, returns false.
4. If `prompt` → shows in-app priming dialog with reason + privacy bullets.
5. User taps CTA → `requestPermission(kind)` triggers OS prompt (web via
   getUserMedia/Notification.requestPermission; native via Capacitor Camera
   / PushNotifications plugins).
6. "Not now" sets `suppressForever(kind)` so we never re-ambush.

Helpers live in `src/lib/permissions.ts`. Already wired into
`VoiceInputButton` (mic) and `AvatarUploader` (photos). Notifications still
use `NotificationPermissionPrompt` (kept for its delayed/iOS-install logic).
Native plugin imports are dynamic so web bundles stay slim.
